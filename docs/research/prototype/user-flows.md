# LifePlanner — User Flows (Trigger Prototype)

> **Role.** Connects the prototype's screens (`docs/research/information-architecture.md` §8.1) into flows and
> checks them against five interaction-completeness criteria: **(1) Happy Path seamless entry→success · (2) every
> exception guides the next action · (3) back / cancel / return-after-leaving paths exist · (4) each screen's states
> (초기/입력중/유효/무효/에러/로딩) are defined · (5) walked as P1, no stuck point.** Grounded in `docs/research/prototype/prd.md`
> (R1–R8) and `docs/core/design-principles.md`. §7 is the re-check + fixes.
> Body English; UI copy Korean. Scope = **prototype** (5 screens); full-app flow is a later superset.

**Core action (= success):** P1 **actually starts** at the moment → **done** (source = execution screen).

> **Note — no network at all.** The prototype is **fully local / on-device (R5)**; it makes **zero** network calls.
> So the entire class of "네트워크 끊김" exceptions **cannot occur** — there is nothing to lose, retry, or sync.
> (Offline is the *default*, not an error state.) This is a deliberate design strength, not an unhandled gap.

## 1. Happy Path (진입 → 성공, 끊김 없이)

```
[첫 실행] 온보딩(왜 실행 알림이 잠금화면을 뚫어야 하는지 설명) → 권한 요청
   │ 허용                                   └ 거부 ─▶ 홈 상단 지속 배너 "실행 알림이 잠금화면을 못 뚫어요 — 켜기"(원탭 설정)
   ▼
[홈=할 일 목록]  초기=빈 상태 "첫 할 일을 정해보자 [+]"
   │ [+ 할 일]
   ▼
[할 일 추가]  필수 시각·제목 │ 실행 알림 ON(기본) │ 선택(접힘): 리드·마이크로노트·단순알림·반복
   │ 저장(유효)              └ 무효(시각 과거/제목 없음)─▶ 인라인 안내, 저장 비활성  └ 취소/뒤로─▶ (입력 있으면) "저장 안 하고 나갈까요?"
   ▼
[홈]  "예정"으로 표시(다음 발화 시각·반복 뱃지)
   ┆   … 다음날 20:30, 잠금+유휴 …
   ▼
[실행의 순간]  FIRING(햅틱) ─▶ COMMIT "어제 네가 21:00 헬스라고 정했잖아" ·[시작할게]
   │ 시작할게                                      └ 30s 무응답/이탈 ─▶ PENDING(조용히 닫힘) ─▶ 다음 열람 캐치업
   ▼
  COUNTDOWN 5·4·3·2·1 ─▶ MICRO "지금 신발 신기" ·[했어→] ─▶ CONFIRM "시작했어?"
   │ 응                         │ 오늘은 못 해              │ 아직/60s
   ▼                           ▼                          ▼
  DONE "안 하던 걸 해냈다"       MISS "괜찮아…데이터야"        PENDING ─▶ 캐치업
   │
   ▼
[홈]  이력에 done(source=실행화면) · 내일 다시(반복)          ◀── ★ 성공. 死角 없이 홈으로 복귀.
```
연속성 확인: 모든 전이가 명시적이고, 어느 상태에서도 **다음 화면/행동이 정의**돼 있으며, 종료(DONE/MISS/PENDING)는 전부
**홈으로 복귀**한다. 막다른 길 없음.

## 2. 주요 분기점
| 지점 | 선택지 | 결과 |
|---|---|---|
| 온보딩 권한 | 허용 / 거부 | 정상 / **우아한 강등**(헤드업+지속 배너) |
| 실행 알림 | ON(기본) / OFF | R2·R3 개입 대상 / 개입 없음 |
| 반복 | 없음 / 매일 / 매주 | occurrence 생성 규칙 |
| 발화 시 기기 | 잠금·절전·킬드 / 전원 OFF | 실행의 순간 발화 / 미발화→캐치업 |
| COMMIT | 시작할게 / 무응답 | 카운트다운 / PENDING |
| CONFIRM | 응 / 아직 / 오늘은 못 해 | DONE / PENDING→캐치업 / MISS |

## 3. 예외 흐름 — 상황 → 처리 → **다음 행동 안내**
> 모든 사용자-대면 예외는 **다음에 무엇을 할지 안내**한다(막히지 않게).

| 상황 | 처리 | → 사용자에게 안내되는 다음 행동 |
|---|---|---|
| 과거 시각 입력 | 무효 | 인라인 "이미 지난 시각이에요" → **시각 고치기**(저장 비활성) |
| 제목/시각 누락 | 무효 | 인라인 힌트 → **채우기**(저장 비활성) |
| 리드가 유효시각을 과거로 | 즉시 발화 | (시스템 처리) 곧 실행의 순간이 뜸 — 사용자 조치 불필요 |
| 동시 발화(같은 시각 2건) | 순차 큐 | 하나 끝내면 **다음 것이 이어서 뜸**(겹치지 않음) |
| 알람 미발화(Doze/OEM) | 자가치유 | 다음 열람 시 **"[제목] 놓쳤어요 — 지금이라도?"** → done/miss |
| 전원 OFF였음 | 미발화→캐치업 | 동일 |
| 실행 중 이탈(잠금/무시/타임아웃) | PENDING | 다음 열람 **"[제목] 아직 안 했죠 — 지금 할까요?"** → done/miss |
| 7일 이상 미해결 | 자동 miss 보관 | 이력에 조용히 기록(다그침 없음) |
| 권한(정확알람/FSI) 거부 | 우아한 강등 | 헤드업 알림 + 홈 지속 배너 **"켜기"**(원탭 설정) |
| **네트워크 끊김** | 해당 없음(로컬 전용) | **아무 영향 없음** — 표시/조치 불필요(오프라인이 기본) |
| 저장 쓰기 실패(희귀) | 재시도 | 토스트 "저장 실패 — 다시" + **재시도 버튼**(입력 유지) |
| 실행 중인 할 일을 삭제/수정 | 안전 처리 | 예약 취소·재예약(고스트 없음); 진행 중 모달이면 닫고 재적용 |

## 4. 뒤로가기 · 취소 · 이탈 후 복귀 (경로 정의)
> 세 가지를 **화면마다** 정의 — 특히 "입력을 잃지 않게"(C2: 도구가 행동보다 무거우면 버려짐).

| 화면 | 뒤로가기 | 취소 | 백그라운드 후 복귀 |
|---|---|---|---|
| **홈** | 앱 나감(루트) | — | 홈 그대로 |
| **할 일 추가/수정** | 입력 있으면 **"저장 안 하고 나갈까요?"** 확인 → 예=폐기, 아니오=계속 | 취소 버튼 = 동일 확인 후 폐기(수정은 원본 유지) | **입력 초안(draft) 복원** — 전화·앱 전환에도 안 잃음 |
| **실행의 순간** | (뒤로 없음 — 모달 순간) 이탈=PENDING | (없음; "오늘은 못 해"가 명시적 이탈) | **캐치업으로 재유입**("아직 안 했죠") — 死角 없음 |
| **캐치업** | 닫기=미룸 | — | 미해결이면 다음 열람에 **앱당 1회** 재노출(나그 없음, R6/R7) |
| **설정** | 홈으로 | — | 변경 즉시 반영·복귀 시 유지 |

> **무효 ≠ 에러**: **무효**=사용자 입력이 규칙에 안 맞음(인라인·복구 쉬움·저장 비활성). **에러**=작업/시스템 실패(재시도 필요).

| **홈** | 빈 상태 CTA | — | 예정+이력 목록 | — | 읽기 실패→재시도 | 진입 시 짧게 |
| **할 일 추가/수정** | 빈 폼(실행알림 ON) | 시각·제목 타이핑 | 시각+제목 有→저장 활성 | 과거 시각·필수 누락→인라인+저장 비활성 | 저장 실패→토스트+재시도(입력 유지) | 저장 중(짧음) |

## 6. Happy Path 연속성 · 안내 완결성 체크
- **끊김?** 없음 — §1의 모든 전이가 명시적이고 종료는 전부 홈 복귀.
- **예외마다 안내?** §3의 모든 사용자-대면 예외에 "다음 행동" 열이 채워짐(네트워크는 해당 없음).
- **뒤로/취소/복귀?** §4에 화면별로 정의(입력 초안 복원 포함).
- **상태 6종?** §5에 무효를 에러와 분리해 정의.

## 7. 재점검 — P1으로 다시 걸으며 (막히는 곳) → 개선
> §1–§5는 아래 발견을 **이미 반영**해 다시 그린 것이다.

1. **첫 실행 권한(치명)** — 맥락 없는 권한 팝업 거부 시 레버가 조용히 죽음 → 온보딩이 *왜*를 먼저 설명, 거부 시 **지속
   배너**로 절대 조용히 실패 안 함.
2. **빈 홈** — 뭘 할지 막힘 → 빈 상태 CTA "첫 할 일을 정해보자".
3. **할 일 추가 부담 + '개입 없는 할 일' 함정** — 필수=시각+제목만, 나머지 접기 · **실행 알림 기본 ON**.
4. **소리 기본 OFF ↔ 못 알아챔** — 프로토타입에서 실측해 확정(지금 미확정).
5. **[신규] 입력 도중 이탈 시 초안 유실** — 계획 세우다 전화 오면 다시 앱 열었을 때 입력이 사라져 P1이 "숙제 같다"며
   이탈[R1-10] → **초안(draft) 복원** + 뒤로/취소 시 **"저장 안 하고 나갈까요?"** 확인(§4).
6. **[신규] 취소/뒤로 경로 부재** — 할 일 추가에서 나가는 명확한 길이 없었음 → **취소 버튼 + 뒤로 확인** 명시(§4).
7. **[신규] 무효 vs 에러 혼동** — 과거 시각(무효)을 시스템 오류처럼 보이면 P1이 뭘 고칠지 모름 → **인라인 무효 안내**로
   "무엇을 고치면 되는지" 그 자리에서 알려줌(§5).

**막히지 않음 확인:** 미스 후 복귀·실행 중 이탈(→캐치업)·오프라인(로컬 전용)·동시 발화(순차) = 死角 없음. ✓
